Linux操作系统下进程间的通信
2022-12-27 21:58:56 #Linux

进程间通信

进程是一个独立的资源分配单元，不同进程间的资源是独立的。但进程不是孤立的，不同进程需要进行信息的交互和状态的传递，进程间通信(IPC, Inter Processes Communication)有以下几个目的:

  1. 数据传输；
  2. 事件通知；
  3. 资源共享；
  4. 进程控制；

终端、进程组、会话与守护进程

终端

UNIX系统中，用户通过终端(如xshell)登录系统后得到一个shell进程，这个终端成为shell进程的控制终端。进程中，控制终端是保存在PCB中的信息，而fork()会复制PCB中的信息，因此由shell进程启动的其他进程的控制终端也是这个终端

进程组

进程组是一组相关进程的合集，进程组由一个或多个共享同一进程组标识符(PGID)的进程组成，进程组的首进程是创建该组的进程，进程组ID即首进程ID

会话

会话是一组进程组的合计，会话首进程是创建该新绘画的进程，其进程ID会成为会话ID。一个会话中的所有进程共享单个控制终端，一个终端最多可能会成为一个会话的控制终端

守护进程(精灵进程)

是linux中的后台服务进程，独立于控制终端(不拥有终端，不受当前控制终端控制，内核无法为守护进程生成任何控制信号)并且周期性地执行某种任务或等待处理发生的某些事件，名字以d结尾。

守护进程的创建步骤

  • 执行一个fork()，之后父进程退出，子进程继续执行

    说明：为了防止进程成为进程组的组长

  • 子进程调用setsid()开启一个新会话

    说明：为了使该进程脱离当前的控制终端，因为当前控制终端是由子进程的父进程所创建，当前会话与父进程ID相同，当前会话与控制终端相连，使用子进程创建一个新的会话，就会脱离当前控制终端

  • 清除进程的umask掩码以确保当守护进程创建文件和目录时拥有所需权限

  • 修改进程的当前工作目录,通常改为根目录

  • 关闭守护进程从其父进程继承而来的所有着的打开着的文件描述符(0,1,2)，使用dup2()将这些描述符重定向到/dev/null

进程间通信的几种方式

根据是否属于同一主机间的进程，通讯方式有两大类：

  • 同一主机进程间通信
    • 匿名管道
    • 有名管道
    • 信号
    • 消息队列
    • 共享内存
    • 信号量
  • 不同主机(网络)进程间通信
    • Socket(套接字)

匿名管道

匿名管道的特点

  1. 管道是一个在内核 内存中维护的缓冲器，这个缓冲器的存储能力是有限的，不同操作系统的管道大小不同

  2. 管道拥有文件的特质，有读/写操作，且管道的读端和写端都各自分配一个文件描述符，匿名管道没有文件实体

  3. 一个管道是一个字节流，在管道中没有消息或消息边界的概念，进程从管道读取数据的大小是任意的，不受写入端数据块大小限制

  4. 通过管道传递的数据是顺序的，管道的数据结构为环形队列

  5. 管道是半双工的，即数据传输的方向是单向的，两个进程在使用管道通讯时，其中一个管道在写入数据，那么另一个进程就无法写入数据

  6. 匿名管道只能在具有公共祖先的进程之间使用。原因是拥有公共祖先的两个进程具有相同的文件描述符表。

由于管道是默认阻塞的，在此情况下管道有以下的读写特点(有名管道与匿名管道的共同特点)：

  1. 当所有指向管道写端的文件描述符被关闭时，如果有进程从管道中读取数据，那么会先读取管道中的剩余数据，当剩余数据被读取完毕之后，再次read()会返回0，就如同读到文件末尾。

  2. 当指向管道写端的文件描述符没有被全部关闭，但持有管道写端的进程并没有向管道中写入数据，那么当其他进程读取数据时，会读取剩余数据，当剩余数据被读取完之后再次read()会阻塞，直到管道中有数据了才会重新读取。

  3. 当所有指向管道读端的文件描述符被关闭时，如果有进程从管道中写入数据，进程会收到一个SIGPIPE信号从而导致进程异常终止。

  4. 当指向管道读端的文件描述符没有被全部关闭，但持有管道读端的进程并没有从管道中读数据，那么另一个进程向管道写数据，一直到管道被写满，再次write()会阻塞。

有名管道

有名管道(FIFO)提供了一个路径名与之关联，以FIFO文件的形式存在于文件系统之中，打开方式与打开一个普通文件的方式一样，能访问该路径的进程，就能够通过FIFO通信。有名管道与匿名管道的不同之处在于：

  1. FIFO在文件系统中作为一个特殊文件存在，但其中的内容存放在内核中的缓冲区，一旦进程结束，这些内容也会消失

  2. 当使用FIFO的进程退出后，FIFO文件仍然继续保存在文件系统中以便下次使用

  3. FIFO有名字，不相关的进程也可以通过它进行通信。

read()函数从文件描述符fd指向的文件中读取count字节的内容到buf所指向的内存空间中，函数返回读取到的字节数，如果读取失败返回-1，返回0代表已经读完，返回值小于count代表本次读取的内容要少于count字节

内存映射(Memory-mapped I/O, mmap)

内存映射是将磁盘文件中的数据映射到内存(进程所分配到的虚拟地址空间)，用户通过修改内存就能修改磁盘文件。

内存映射的使用

1. 功能：在内存中创建一个磁盘文件的映射
2. 参数：

  • void *addr: NULL, 由内核来指定

  • size_t length: 要映射到内存中的数据的长度,值不能为0，建议文件有多长，就映射多长

    • 获取文件的长度: stat(),lseek()

  • int prot：对申请的内存映射区的操作权限

    • PROT_EXEC： 可执行的权限
    • PROT_READ: 可读权限
    • PROT_WRITE：可写权限
    • PROT_NONE: 无权限

要操作映射内存，必须要有读的权限

  • int flags:

    • MAP_SHARED: 映射区的数据会自动和磁盘文件同步，进程间通信则必须设置
    • MAP_PRIVATE: 不同步，映射区的数据改变了，原来的文件不会改变(copy on write)

  • int fd: 需要被映射的文件的文件描述符，通过open()得到，打开一个磁盘文件，文件的大小不能为0，打开时指定的权限不能和prot参数有冲突

  • off_t offset: 距离文件开始处的偏移量(4k的整数倍)，一般为0，表示不偏移

3. 返回值： 成功则返回映射的内存地址；失败返回MAP_FAILED,即(void *) -1

1. 功能：释放内存映射
2. 参数：

  • void *addr: 要释放的内存地址
  • size_t length: 释放内存的大小
  1. 返回值: 成功返回0，失败返回-1
使用内存映射实现进程间的通信
  1. 父子进程：

  • 通过唯一的父进程先创建内存映射区

  • 创建内存映射区之后，创建子进程

  • 父子进程共享内存映射区

  1. 无关进程

  • 准备一个大小不是0的磁盘文件

  • 进程1通过磁盘文件创建内存映射区，得到操作该块映射区的指针

  • 进程2通过磁盘文件创建内存映射区，得到操作该块映射区的指针

  • 使用内存映射区通讯

内存映射区通信是非阻塞的

  2. 创建一个新文件，拓展该文件(使其大小不为0)

  3. 把新文件的数据映射到内存中

  4. 通过内存拷贝将第一个文件的内容拷贝到第二个文件中

匿名映射

指不需要文件实体进行的一个内存映射，用于父子进程。将mmap()中的MAP_SHARED更改为MAP_ANONYMOUS,设置后，文件描述符会被忽略，偏移量必须为0

内存映射的注意事项

  1. 可以对mmap()的返回值ptr做++操作，但此时munmap()会失败，因此建议在++之前保存原始地址
  2. 如果文件open的权限为O_RDONLY，mmap()时prot参数制定为PROT_READ | PROT_WRITE，此时mmap()会映射失败，**返回MAP_FAILED**，prot()对内存映射区的权限应当小于等于源文件open()时的权限
  3. mmap()创建映射区时的文件偏移量应当为4096的整数倍，若为其他值，也会返回MAP_FAILED
  4. mmap()在第二个参数为0时，也会调用失败，返回MAP_FAILED
  5. open()时可以O_CREAT一个新文件来创建映射区，但是需要保证文件大小不为0，可以使用lseek()，truncate()来对文件进行扩展
  6. mmap()后关闭文件描述符对mmap映射没有影响，因为在mmap()调用时，相当于拷贝了一份文件描述符

共享内存(最快，效率高)

共享内存允许两个或者多个进程共享物理内存的同一块区域(通常被称为段)。由于一个共享内存段会称为一个进程用户空间的一部分，因此这种通信机制无需内核介入。所有需要做的就是让一个进程将数据复制进共享内存中，并且这部分数据会对其他所有共享一个段的进程可用

使用步骤

  • 调用shmget()创建一个新的共享内存段或者取得一个由其他进程创建的共享内存段的标识符。这个调用返回后续所要使用的共享内存标识符

    • 功能：创建一个新的共享内存段，或者获取一个既有的共享内存段的标识，新建的共享内存段中的参数都会初始化为0

    • 参数：

      • key：ket_t是一个整形，16进制非0值表示，通过这个值找到或创建一个共享内存，可以随便指定，也可以由ftok()函数来生成

      • size：共享内存的大小

      • shmflg：共享内存的属性，如访问权限，创建\判断共享内存是否存在

        IPC_CREAT：创建

        IPC_EXCL：判断共享内存是否存在，需要和IPC_CREAT一起使用

    • 返回值：成功返回共享内存的shmid，操作这块内存都是用这个值，失败返回-1

  • 使用shmat()来在调用进程的虚拟内存中附上共享内存段，该函数返回指向进程虚拟地址空间中该共享内存段的起点的一个指针addr

    • 功能：将内存和当前进程关联

    • 参数：

      • shimid：共享内存的标识，由shmget()的返回值得到

      • shmaddr：共享内存的地址，设置为NULL，由系统指定

      • shmflg：进程对共享内存的操作权限

        SHM_RDONLY：只读权限，必须要有读的权限

        0：读写权限

    • 返回值：成功返回共享内存在虚拟内存中的首地址，失败返回(void *) -1

  • 使用shmdt()来分离共享内存段，在进程终止时，也会自动调用

    • 功能：解除当前进程和共享内存的关联

    • 参数：共享内存的首地址

    • 返回值：成功返回0，失败返回-1

  • 使用shmctl()来删除共享内存段。只有当所有使用该共享内存段的进程都与之分离之后内存段才会被销毁，只有一个进程需要调用该函数

    • 功能：对共享内存进行操作

    • 参数：

      • shmid：要删除的共享内存id

    • cmd：所要做的操作
      + IPC_STAT：获取共享内存当前状态

      • IPC_SET：设置共享内存的状态
        • IPC_RMID：标记共享内存需要被销毁

    • buf：需要设置或者获取的共享内存属性信息

      • 若设置IPC_STAT：buf中存储相关数据

      • 若设置IPC_SET：buf中需要设置数据

        • 若设置IPC_RMID：buf无用，NULL

      • 返回值：成功返回共享内存引用的id，操作这块内存都是用这个值，失败返回-1

  • 其他函数介绍：

    • 功能：根据指定的路径名，生成一个ket_t参数
    • 参数：
      • pathname：文件的路径，该路径必须是已经存在的路径
      • proj_id：int类型的值，系统调用只会使用第一个字节，一般指定一个字符’a’

一些问题

  • 共享内存和内存映射的区别

    1. 共享内存可以直接创建，内存映射需要磁盘文件(匿名映射除外，但只能用于父子进程通信)
    2. 共享内存效率更高
    3. 共享内存是所有进程操作同一块内存，内存映射时，不同的进程在各自的虚拟地址空间中有一个独立的内存
    4. 数据安全：
      • 进程突然退出时，共享内存还存在，而内存映射区就消失了
      • 当运行进程的电脑死机、宕机时，共享内存中的数据就会丢失，而磁盘文件中的数据依然存在，内存映射区的数据可以恢复
    5. 生命周期：
      • 进程退出，内存映射区销毁
      • 进程退出，共享内存还在，与共享内存的关联自动断开，只有所有的关联进程数为0时共享内存才销毁

  • 可不可以对共享内存进行多次删除？

    答：可以的，因为shmctl只是标记删除共享内存，不是直接删除，当和共享内存关联的进程数为0时，他就真正被删除了，当共享内存的key值被标记为0时，代表这块共享内存被标记删除

  • 操作系统如何知道一块共享内存被多少个进程关联？

    答：共享内存维护了一个名为shmid_ds的结构体，其中一个成员shm_nattch记录了一个共享内存关联的进程个数

信号量

信号是事件发生时对进程的通知机制，有时也称为软件中断，是软件层次上对中断机制的一种模拟，是一种异步通信，可以使一个正在运行的进程被另一个正在运行的异步进程中断，转而处理一个突发事件。(使用kill -l 查看所有的信号)

发往进程的信号大多都源于内核：

  • 对于前台进程，输入Ctrl +Ｃ通常会给进程发送一个中断信号
  • 硬件发生异常时，硬件检测到一个错误条件通知内核，内核发送相应信号给相关进程
  • 系统状态变化，如定时器引起的SIGALRM信号
  • 运行kill命令或调用kill函数

信号的特点：

  • 简单
  • 不能携带大量信息
  • 满足特定条件才能发送
  • 优先级高

几个重要的信号：

  • SIGINT (2)：按下ctrl+c时终止由该终端启动的正在运行的进程(默认处置：term)
  • SIGQIUT (3)：按下ctr+\时终止由该终端启动的正在运行的进程
  • SIGKILL (9)
  • SIGSEGV (11)：指示进程进行了无效访问，终止进程并产生core文件(默认处置：core)
  • SIGPIPE (13)：管道破裂，进程终止
  • SIGCHILD (17)：
    • 子进程结束时，父进程会收到该信号
    • 子进程收到SIGSTOP信号停止时
    • 子进程处于停止状态，接收到SIGCONT后唤醒时
    • 父进程默认忽略SIGCHILD信号，但可以捕获SIGCHILD信号以解决僵尸进程问题
  • SIGCONT (18)：使进程继续运行(默认处置：cont)
  • SIGSTOP (19)：暂停进程的执行(默认处置：stop)

信号的的三种状态有:产生,未决,递达

信号的使用

  • 功能：给某个进程pid发送信号sig

  • 参数：

    • pid：进程号

      pid>0,将信号发送给指定的进程；pid=0，将信号发送给当前进程组；pid=-1，将信号发送给每一个有权限接收这个信号的进程；pid<-1，这个pid就等于某个进程组的id取反

信号集

是一种数据结构sigset_t，表示多个信号的集合。在信号控制块(PCB)中有两个重要的信号集：阻塞信号集与未决信号集。sigset_t是一个64位的整数，每一位的0/1代表了对应信号的一个状态，比如阻塞信号集的第一位为1，则代表1号信号为阻塞状态

信号的”阻塞”指的是阻止信号被处理，但是不阻止信号产生；

示例：

  1. 用户通过键盘Ctrl + c产生2号信号SIGINT (信号被创建)
  2. 当信号没有被处理时，信号处于未决状态，将储存在未决信号集中，二号位标记为1，表示该信号未决
  3. 未决状态的信号，需要和阻塞信号集中对应的标记位比较，若在阻塞信号集上该信号标记为1(阻塞状态)，那么暂时不处理该信号，继续处于未决状态。阻塞信号集默认不阻塞任何信号，需要调用系统API进行阻塞设置

  • 功能：清空信号集set中的数据，所有标志位置0

  • 参数：set为传出参数，代表需要操作的信号集

  • 返回值：成功返回0，失败返回-1

  • 功能：标记信号集set中的数据，所有标志位置1
  • 参数：set为传出参数，代表需要操作的信号集
  • 返回值：成功返回0，失败返回-1
  • 功能：设置信号集中的某一个信号对应的标志位1，表示阻塞这个信号
  • 参数：set为传出参数，代表需要操作的信号集；signum代表需要设置阻塞的信号
  • 返回值：成功返回0，失败返回-1
  • 功能：设置信号集中的某一个信号对应的标志位0，表示不阻塞这个信号
  • 参数：set为传出参数，代表需要操作的信号集；signum代表需要设置非阻塞的信号
  • 返回值：成功返回0，失败返回-1
  • 功能：查询某个信号是否被阻塞或未决
  • 参数：signum代表需要查询的信号
  • 返回值：1表示signum被阻塞或未决，0表示未被阻塞或已被解决
  • 功能：获取当前进程中的未决信号集
  • 功能：检查或者改变PCB中的信号集
  • 参数：
    • how：
      • SIG_BLOCK：将用户设置的阻塞信号集添加进内核中，内核中的信号集是mask，那么内核实现的方式是：mask | set
      • SIG_UNBLOCK：根据用户设置的信号集分布，对内核中的信号进行解除阻塞，内核中的信号集是mask，那么内核实现的方式是：mask & ~set
      • SIG_SETMASK：覆盖内核中原来的值
    • set：已经初始化好的用户的自定义信号集
    • oldset：保存设置之前的内核中的阻塞信号集状态，可以是NULL
  • 返回值：成功返回0，错误返回-1

内核实现信号捕捉的过程

在执行主程序的某条指令因为中断、异常或者系统调用进入内核—>内核处理完异常之后处理当前进程中是否由可以递送的信号do_signal()—>若信号的处理动作为自定义的信号处理函数，则回到用户区执行处理函数—>处理函数执行完毕之后通过系统调用sigreturn再次回到内核—>再从内核中回到上次主程序中断处继续执行。

  • 在回调函数执行时，会使用临时阻塞的信号集，当回调结束之后，使用内核中的阻塞信号集
  • 当一个信号正在被处理时，系统又收到一次该信号，那么本次信号会默认被屏蔽
  • 阻塞的常规信号(前32个)不支持排队，即同一个信号同时接收多次，当信号在被处理之前，该信号位只被标记一次
